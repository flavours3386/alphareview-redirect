# alphareview-redirect - Product Sense

## 미션

인앱 브라우저 환경에서도 알파리뷰 프로모션 신청 폼에 정상 접근할 수 있도록, 외부 브라우저 전환을 자동화한다.

## 해결하는 문제

- 카카오톡/네이버 등 인앱 브라우저에서 Tally 폼이 정상 렌더링되지 않음
- 알림톡/LMS CRM 메시지의 CTA 링크가 인앱 브라우저에서 열림
- iOS 인앱 브라우저는 외부 전환 API가 없어 사용자가 수동 전환해야 함

## 사용자

| 사용자 | 역할 | 주요 시나리오 |
|--------|------|-------------|
| 프로모션 대상 고객 | 아임웹 쇼핑몰 운영자 | 카카오톡 알림톡에서 프로모션 신청 링크 클릭 |
| 세일즈 AE | CRM 메시지 발송 | CTA 링크에 리다이렉트 URL 삽입 |

## 핵심 가치

1. **투명한 전환**: 사용자가 "리다이렉트 중"임을 인지하면서 자연스럽게 외부 브라우저로 이동
2. **최대 호환**: 카카오톡, 네이버, LINE, Instagram, Facebook, Twitter, SamsungBrowser 커버
3. **폴백 안전망**: 자동 전환 실패 시 수동 버튼으로 100% 도달 보장

## 제품 원칙

- 사용자 경험 최우선: 일반 브라우저에서는 리다이렉트 페이지가 보이지 않아야 함 (즉시 이동)
- 기술 부채 최소화: 단일 HTML 파일, 외부 의존성 제로
- 에러 상태 명확: 잘못된 링크일 때 명확한 메시지 표시

## 성공 지표

| 지표 | 정의 | 목표 |
|------|------|------|
| 외부 전환 성공률 | 리다이렉트 → Tally 폼 도달 비율 | 90% 이상 |
| 일반 브라우저 지연 | 리다이렉트 페이지 노출 시간 | 1초 미만 |
| 폴백 버튼 사용률 | 수동 버튼 클릭 비율 (낮을수록 좋음) | 10% 미만 |

## 경쟁 대안

| 대안 | 장점 | 단점 | 선택하지 않은 이유 |
|------|------|------|-------------------|
| Tally 직접 링크 | 간단 | 인앱 브라우저 미지원 | 핵심 문제 해결 불가 |
| 자체 서버 리다이렉트 | UA 분석 정밀 | 서버 비용, 유지보수 | 과잉 엔지니어링 |
| Branch.io 딥링크 | 고급 분석 | 유료, 설정 복잡 | 단순 리다이렉트에 과잉 |
